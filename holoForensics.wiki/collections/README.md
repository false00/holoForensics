# Collection Index

This index tracks the collection contracts known to the current Rust runtime.

The desktop UI also presents a broader Windows forensic collection portfolio for demo and planning use. That UI surface intentionally includes additional evidence sources beyond the active runtime contracts below. Unless a collection appears in this index or under `src/collections/`, treat it as a desktop brief rather than a live collector implementation.

## Required Updates

- Keep this index aligned with `src/collection_catalog.rs`.
- Add a new page here whenever a new collection contract or collector implementation is enabled.
- Update `Status` when the collector implementation mode changes.
- Update the affected collection page whenever a major collector behavior, acquisition method, output layout, manifest schema, or validation status changes.
- New live collectors that write per-collector metadata into a collection zip must use the centralized archive layout from `src/collection_metadata.rs`: `$metadata/collectors/<volume>/<collector>/manifest.json`. If the collector writes an operator-facing collection log, place it beside the manifest as `collection.log`.
